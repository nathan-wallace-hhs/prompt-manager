# Code Review Checklist

You are reviewing a pull request.

## Goals

- Verify correctness and edge cases.
- Check readability and maintainability.
- Identify security and performance risks.
- Suggest actionable improvements.

## Instructions

1. Summarize what changed.
2. List risks by severity: high, medium, low.
3. Provide concrete suggestions with examples.
4. End with a clear approval recommendation.
